## 📖 Day 25-27 详细规划：AI内容创作平台

### 🎯 项目概述

**AI内容创作平台**是一个多Agent协作的内容生成系统，类似于一个"AI编辑部"。用户输入主题或需求，多个专业Agent协作完成从选题、研究、写作到审核的完整内容生产流程。

**核心功能**：
- 🎨 **多Agent协作** - 选题Agent、研究Agent、写作Agent、审核Agent分工协作
- 📝 **内容生成** - 支持文章、博客、社交媒体文案等多种内容类型
- 🔍 **智能研究** - 自动收集和整理相关资料
- ✍️ **风格定制** - 支持不同写作风格（专业、轻松、营销等）
- 📊 **内容管理** - 生成历史、版本控制、导出功能

**技术栈**：
| 层级 | 技术 |
|------|------|
| 前端 | Streamlit |
| API | FastAPI |
| Agent框架 | LangChain + 自定义Agent |
| 数据库 | SQLite + SQLAlchemy |
| 工作流 | 自定义工作流引擎 |
| LLM | OpenAI API / DashScope |

| **优化Agent** | 优化内容结构和表达 | 初稿 | 优化稿 |
| **审核Agent** | 检查内容质量和合规性 | 优化稿 | 最终稿+评分 |

### 📅 Day 26: 后端API与工作流

#### 🎯 学习目标
- 设计RESTful API接口

- 添加任务队列支持

#### 📝 详细内容

**1. API设计**

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/v1/content/generate` | 生成内容 |
| GET | `/api/v1/content/{id}` | 获取内容 |
| GET | `/api/v1/content/` | 列表查询 |
| PUT | `/api/v1/content/{id}` | 更新内容 |
| POST | `/api/v1/content/{id}/regenerate` | 重新生成 |
| POST | `/api/v1/content/{id}/review` | 提交审核 |
| GET | `/api/v1/tasks/{id}` | 查询任务状态 |

**2. 工作流引擎**
```python
class ContentWorkflow:
    """内容生成工作流"""
    
    def execute(self, requirement: str) -> Content:
        # 1. 需求分析
        req_result = self.requirement_agent.analyze(requirement)
        
        # 2. 并行执行研究和写作
        with ThreadPoolExecutor() as executor:
            research_future = executor.submit(
                self.research_agent.research, 
                req_result.keywords
            )
            # 等待研究完成后再写作
            research_result = research_future.result()
            
        # 3. 写作
        draft = self.writing_agent.write(req_result, research_result)
        
        # 4. 优化
        optimized = self.optimization_agent.optimize(draft)
        
        # 5. 审核
        final = self.review_agent.review(optimized)
        
        return final
```

**3. 内容审核机制**
- 自动审核：敏感词过滤、质量评分
- 人工审核：标记待审核内容
- 审核状态：pending → approved → rejected

**4. 任务队列**
- 使用 BackgroundTasks 处理异步生成
- 任务状态跟踪：queued → processing → completed → failed
- 支持任务取消和重试

**5. 学习要点**
- RESTful API设计原则
- 异步任务处理

#### 📝 详细内容

**1. 为什么测试很重要？**

| 没有测试 | 有测试 |
|---------|--------|
| 改代码担心出bug | 改完就能验证 |
| 手动测试费时费力 | 一键运行全部测试 |
| 新人不敢改老代码 | 测试用例就是文档 |
| 上线前提心吊胆 | CI 自动检查 |

**2. pytest 核心概念**

```python
# 测试函数 - 命名以 test_ 开头
def test_add_document():
    service = DocumentService(db)
    doc = service.create_document("测试", "内容")
    assert doc.title == "测试"

# Fixture - 共享测试数据
@pytest.fixture
def db_session():
    engine = create_engine("sqlite:///:memory:")
    Base.metadata.create_all(engine)
    session = Session(bind=engine)
    yield session
    session.close()

# Mock - 模拟外部依赖
from unittest.mock import Mock, patch

@patch("knowledge_base.services.llm_service.call_llm")
def test_with_ai(mock_llm):
    mock_llm.return_value = "模拟的AI回复"
    result = service.answer_question("你好")
    assert result == "模拟的AI回复"
```

**3. 测试分类**

| 类型 | 速度 | 范围 | 示例 |
|------|------|------|------|
| **单元测试** | 很快 | 单个函数/类 | 测试文档创建逻辑 |
| **集成测试** | 较快 | 多个组件 | 测试API请求-响应 |
| **端到端测试** | 慢 | 完整流程 | Selenium自动化 |

**4. 测试文件结构**


#### 📝 详细内容

**1. 为什么需要结构化日志？**

```
❌ 普通日志（难以搜索和分析）：
[ERROR] 2024-01-15 10:30:45 API调用失败 connection timeout

✅ 结构化日志（JSON格式，可搜索）：
{
  "timestamp": "2024-01-15T10:30:45Z",
```

```python
import structlog
import logging

def configure_logging():
    """配置结构化日志"""
    structlog.configure(
        processors=[
            structlog.stdlib.add_log_level,
            structlog.processors.TimeStamper(fmt="iso"),
            structlog.processors.JSONRenderer(),
        ],
        wrapper_class=structlog.stdlib.BoundLogger,
        logger_factory=structlog.stdlib.LoggerFactory(),
    )

logger = structlog.get_logger()

# 使用
logger.info("content_generated", 
    content_id="abc123", 
    duration=2.5
)
```

**3. 请求链路追踪**

```python
import uuid
from fastapi import Request

@app.middleware("http")
async def add_trace_id(request: Request, call_next):
    """添加请求追踪ID"""
    trace_id = request.headers.get("X-Trace-ID", str(uuid.uuid4()))
    
    # 绑定到当前请求上下文
    structlog.contextvars.bind_contextvars(trace_id=trace_id)
    
    response = await call_next(request)
    response.headers["X-Trace-ID"] = trace_id
    
    return response
```

```python
        for v in checks.values()
    )
```
